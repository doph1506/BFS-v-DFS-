Installation

Even if you have purchased only some of the PowerDesigner modules, you can install the others in stand-alone mode for a 15 days evaluation period.

Non-administrator users may receive a warning message during installation. Setup requires write access on certain directories and on the registry that non-administrator users may not have. Setup can continue despite this warning message but will fail if the user does not have the required rights.

PowerDesigner supports the Sybase Asset Management (SYSAM) system used by other Sybase products, which provides various forms of license. You can install PowerDesigner in trial mode, or with a stand-alone local, stand-alone served, or floating license. In stand-alone served and floating mode, be sure to indicate both server name and port. Even if optional, it is better to indicate the port number, the default value for which is 27000.

Installation on Windows 2000 requires that all Windows security patches are installed.
In particular, you need to install a specific security fix (Security Update for Windows 2000 KB835732) which is a requirement for .Net 2.0 sp1 installed by PowerDesigner.

Installation alongside previous versions of PowerDesigner

PowerDesigner 15.2 will automatically update any instance of PowerDesigner 15.0 or 15.1.

PowerDesigner Portal is the new name for the PowerDesigner Repository Web Browser. Before installing the PowerDesigner Portal 15.2 you must uninstall the PowerDesigner Repository Web Browser 15.0. Upgrades from an existing 15.1 installation are supported.

The first time you open version 15.1 or lower models in PowerDesigner 15.2, they will be updated to version 15.2. This process takes longer than the standard time needed to open a model saved with version 15.2

Models saved with version 15.2 may be corrupted if they are opened in version 15.1 or lower.

  • Customized resources and repository
    All customized resources must be opened and saved in PD 15.1 or above to be properly recognized as resource files by the repository.

  • Connection to a Unicode database
    For some databases, you may have to set up charset information in the ODBC drivers dialog when you connect to a Unicode database.
    Depending on the character set that the target database uses, you may have to change the charset attribute of your ODBC data source.
    For example, to connect to a Unicode Sybase Adaptive Server Enterprise database, you must indicate in the charset field the code of the Unicode character set (in this case "utf8").
    Each ODBC driver has its own tuning parameters to indicate the character set. This information is located in the driver help file. Search for the keywords "charset" or "character set" in the ODBC driver help file.

  • "Not Certified" resource files
    Some resource files are delivered with "Not Certified" in their names.
    Sybase will perform all possible validation checks, but we do not maintain specific environments to fully certify these resource files. We will support them by accepting bug reports and providing fixes as per standard policy, with the exception that there will be no final environmental validation of the fix. You are invited to assist us by testing fixes and reporting any continuing inconsistencies.

Additional information and known issues

  • Extensions
    All extensions have to be resaved to take advantage of SAM selection

  • Symbols changes
    More symbols are generic now and you may see some size changes when you open models saved with previous releases.

  • J# redistributable
    PD redistributes MS Visual J# Redistributable Package 2.0 for use in the autolayout function
    If J# is removed, PD can still be used but only a basic autolayout feature is available.

  • PowerDesigner installed on a network drive
    When PowerDesigner is run over a network, you must authorize network execution before running it on a client computer.
    Use the caspol executable located in the Microsoft.net directory and run the following command
          caspol -m -ag LocalIntranet_Zone -url \\<ServerName>\<FolderName>\* FullTrust -n "<Name>" -d "<Description>"

    You must run PowerDesigner in administration mode (even if your user is an administrator) in order to be able to install a plug-in through Tools > General options > Add-ins.

  • PowerDesigner project and file system
    PowerDesigner project content is not synchronized with the file system. It is a logical view of the project independent of the files' locations.
    When used as a plug-in in Eclipse or Visual Studio, PowerDesigner project content is not synchronized with project content in Eclipse or Visual Studio.

  • Changes in metamodel and API since 15.0

    • Removals BasePhysicalPackageOptions.ReverseCountTableRow BasePhysicalPackageOptions.DefaultOwners
    • Better access to comparison in VBS
      You can now create a helper class with a method CreateComparisonHelper(newobject) available on models and targets.

      This class currently presents two methods:
         int GetDifferencesCount()
         bool GetDifferenceInformation(in int pos, out int diffKind, out object OldObject, out object NewObject, out string MemberPublicName, out string OldValue, out string NewValue)

      Example:
         Set M1 = OpenModel("c:\temp\ConceptualDataModel_1.cdm")
         Set M2 = OpenModel("c:\temp\ConceptualDataModel_2.cdm")
         Set cmp = M1.CreateComparisonHelper(M2)
         for i = 0 to cmp.GetDifferencesCount() - 1
             dim oldObj, newObj, diffKind, member, oldVale, newVale
             cmp.GetDifferenceInformation i, diffKind, oldObj, newObj, member, oldVale, newVale
             dim oldDsc, newDsc
             oldDsc = "?"
             if not oldObj is nothing then
                oldDsc = oldObj.ShortDescription
             end if
             newDsc = "?"
             if not newObj is nothing then
                newDsc = newObj.ShortDescription
             end if
             output "Diff " & Cstr(i) & " old=" & oldDsc & " new=" & newDsc & " member=" & member & "
         old=" & oldVale & " new=" & newVale
         next
         cmp.Delete
         set cmp = nothing

    • Replacement of a shortcut by a copy of the source object
      You can use this method on any shortcut to remove the shortcut, replace it by a copy of the source object, and reconnect all links properly.

      Use the method Shortcut.ReplaceByCopyOfTargetObject

    • Check out any repository version of a document
         RepositoryBaseDocument.CheckOutOldVersionToFile

      Example:
         Set D = RepositoryConnection.childObjects.Item(1)
         D.CheckOutOldVersionToFile "2.1.1", "c:\temp\oldversion.pdm"

    • Get the list of child objects having a stereotype
      Use new Method ExtensibleObject.GetCollectionByStereotype

      Example :
         Set collection = ActiveModel.GetCollectionByStereotype("RuleStereotype")
         Set newRule = collection.CreateNew() ' Creates a stereotyped business rule
         ' Works also with GetCollectionByName but requires << >>
         Set collection = ActiveModel.GetCollectionByName("<<RuleStereotype>>")

    • Research an object by name or code
      Use the method NamedObject.FindChildByName & FindChildByCode

      Example
         Set 1rstFound = ActiveModel.FindChildByName("Partition_10", Cls_ExtendedObject, "Partition")
         ' last parameter is the optional stereotype
         Set 2ndFound = ActiveModel.FindChildByName("Partition_10", Cls_ExtendedObject, "Partition", 1rstFound) ' search another one with same name

    • Change the custom shape of a symbol
      Set the attribute CustomPictureFile of SolidSymbol to a path inside picture library

      Example

         dim sym
         for each sym in ActiveDiagram.Symbols
            sym.CustomPictureFile = "piclib://Enterprise Architecture.piclib#Business\Balance.png"
         next

      This works for all kinds of pictures (files, piclib, office, DLL, ...).
      For piclib & office, the URL syntax is independent of file location, which makes it easier to use the same VBScript on different machines.
      If you want to know the URL syntax for a given picture, you can copy the URL in the Change Image dialog: the name field value can be copy/pasted.



  • Connection profile limitations
    Supported and tested native drivers are the following:
    • Sybase ASE 12.5 and 15
    • Oracle 9, 10 and 11
    • Microsoft SQlServer 2000, 2005 and 2008
    • JDBC (Sybase ASE drivers)

    Support for Direct Connect, ADO and OLE DB connection have been removed from PD 15.x.

    Reverse engineering cannot be performed through JDBC for the following databases:
    • Microsoft Access 2000
    • Microsoft Access 95 & 97 (Deprecated)
    • ADABAS D
    • ALLBASE/SQL G.1 (Not Certified)
    • ANSI Level 2
    • IBM DB2 5.x for OS/390 (Deprecated), UDB 5.x Common Server (Deprecated), UDB 6.x Common Server (Not Certified)
    • INFORMIX SQL 8.x (Not Certified)
    • MySQL 3.22 (Deprecated), 3.23 (Deprecated), 4.0
    • NonStop SQL (Not Certified)
    • ODBC 3.0
    • Microsoft SQL Server 7.x (Deprecated)

    A JDBC connection requires that the path to the bin directory of JRE is included in the PATH environment variable.
    If you must connect using JDBC to several databases using different JDBC drivers, the path to all JDBC drivers must be added to Classpath.

  • Change target DBMS Oracle 10 to Oracle 11.
    The implementation of cluster indexes in the Physical Data Model has changed between Oracle 10 and Oracle 11. In order to maintain cluster index definition, you must not use the Change Target feature to transform an Oracle 10 PDM into an Oracle 11 PDM. You must instead use a PDM to PDM generation and allow transformations (in the Detail tab).

    The index cluster will be transformed into the new implementation during this process.

  • Change in user-defined type representation for Microsoft SQLServer
    In PowerDesigner 12.5 and before, Abstract Data Types could be used to generate aliases to basic types or CLR types. They can now only be used to create CLR types, and aliases must be created using Domains.

  • Checking out 12.5 logical model from repository
    The checking-out of Logical 12.5 PDM will no longer convert the PDM into LDM. The conversion is now done during the first opening after checkout.

  • Reverse engineering of Where clause from Replication Server in the Information Liquidity Model
    For Replication Server 12.5 or 12.6, the reverse engineering of article and subscription "where" clauses can be incomplete if the columns involved in these clauses have the following data types (the result is different depending on whether RSSD is deployed on an ASA or ASE server):
    ASA RSSD:
    • smallmoney
    • datetime
    • date
    • time
    • smalldatetime
    • numeric
    • decimal
    • float

    ASE RSSD:
    • numeric
    • decimal


  • DBMS-Specific Physical option tab in Physical Data Model
    In some cases, the new DBMS-specific tab for physical options in the PDM will not show all possible physical options. If you need to use one of the missing options, you must use the standard physical options tab.

  • Issues with ASE ODBC Driver and PD repository
    When using an ASE database for the PD repository with an ASE ODBC driver, you must select the "Enable Dynamic Prepare" option on the "Connection" tab.

    When using an ASE 15 database for the PD repository with an ASE ODBC driver, documents consolidated will be extracted and truncated per package of 32k each section of 1 MB.
    For example:
    • PT (1 165 KB); size after extraction = 64 KB
    • JPG (305 KB); size after extraction = 32 KB
    • DOC. (2 803 KB); size after extraction = 96 KB
    • PDF (4 695 KB); size after extraction = 160 KB
    • ZIP (6 574 KB); size after extraction = 224 KB
    • PDF (13 646 KB); size after extraction = 448 KB

    Work around is to increase the value of the parameter Text Size on Advanced page of the ODBC driver ASE 15.

  • Drag/drop from browser to diagram
    Drag/drop from browser to diagram will always create a shortcut. The general option "Drag/drop default action" that was applied in version 12.0 and before is now only used in drag/drop targeting the browser.
    If you actually want to move an object by drag/drop targeting a diagram, you must use the key combination Shift + Drag/Drop

  • List report
    Multicolumn sort is not available. Sort uses only the first column.

  • HTML template for list report
    List reports generated in HTML format are based on a template. The interface does not allow you to easily change this template. The template used is the default one used for reports.
    In order to modify the HTML template used when generating a list report in HTML format, you must modify the default template for Report.
    Open the property sheet of any report, select the template you want to use, click the button "Set as default" and cancel.

  • Specific behavior under Windows Vista
    The Execute command may not be able to run commands properly unless PowerDesigner has been launched in Administration mode under Vista.

  • License server and Vista
    The license server, Flexlm, cannot be run as a service under Vista. You must run it as an executable by double-clicking on it in a file explorer.

  • Limitations on Windows 64-bit platforms
    Although PowerDesigner is not officially supported on Windows 64-bit platforms, PowerDesigner can run on these platforms.
    Setup will not install the x64 version of .Net framework 2.0 SP1. You should install .Net framework 2.0 SP1 (x64) separately to be able to use the following PowerDesigner features : auto-layout, import/export XMI, import Microsoft Visio.

  • Limitations on Windows 2000 platforms
    The new improved RTF support is not available on Windows 2000 platforms.

  • License server and PowerDesigner on same computer
    You cannot run PowerDesigner on a computer where a license server is installed.
    If you are using a stand-alone served or a mobile floating license and you work disconnected from the server, and if your workstation hosts other products that also use Flexlm served licenses, you may have problems when returning your license to the server.

  • Dual Monitor
    When using dual monitors, if you want PowerDesigner to use the last monitor where you have placed a property sheet to display the next one, you will have to set up the following registry key :
         [HKEY_LOCAL_MACHINE\SOFTWARE\Sybase\PowerDesigner 15\General]
         "UseLastActivePopupMonitor"="yes"

  • Repository Portal
    The JDBC driver you select in setup or in the Database connection page in configuration parameters must be compatible with the JDK installed on your computer.

    Connection to Microsoft SQLServer requires a unified Microsoft JDBC driver that supports connections to all versions of Microsoft SQLServer and can be downloaded from http://www.microsoft.com/downloads/details.aspx?FamilyID=99b21b65-e98f-4a61-b811-19912601fdc9&displaylang=en.

The OLE Automation folder in the PowerDesigner directory contains some samples for using OLE automation to access PowerDesigner objects.

Repository

PowerDesigner 15.2 is delivered with an authenticated Sybase SQL Anywhere 11.0.0 database, which can be used as a PowerDesigner repository database.

Please refer to the PowerDesigner Installation Guide or to online help (search for Authenticated keyword) for the installation procedure. The Sybase SQL Anywhere engine delivered with PowerDesigner cannot be used for any purpose other than a PowerDesigner repository database.

Importance of time setting

PowerDesigner uses modification time comparison in various functions. Be sure that all computers installed with PowerDesigner are set with the accurate time.
